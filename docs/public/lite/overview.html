⚙ Stack

Technology Stack

Claude Code의 핵심 기술 스택과 선택 근거입니다.

Component Technology Why
Language TypeScript 도구 스키마와 API 타입 안정성
Runtime Bun 빠른 콜드 스타트, 네이티브 HTTP 스택
Terminal UI React + Ink 선언형 터미널 UI 구성
Bundler Bun (built-in) 단일 파일 번들 및 소스 맵 생성
Native Layer Zig HTTP 전송과 암호화 해시 처리
Feature Flags GrowthBook 배포 없이 원격 설정 및 테스트
Telemetry OpenTelemetry 분산 추적으로 성능과 오류 모니터링
State React Hooks + Context 세션 및 UI 상태 관리

★ Insights

Key Discoveries

소스 분석에서 발견된 주요 설계 결정들입니다.

🔍
Prompt-Based Orchestration 에이전트 오케스트레이션이 코드가 아닌 프롬프트로 구현되어 있습니다. 코드 변경 없이 전략을 수정할 수 있도록 설계되었습니다.
⚠️
KAIROS: 자율 데몬 모드 완전히 구현되어 있지만 비활성화된 자율 에이전트 데몬입니다. 주기적으로 작업 상태를 평가하고 유휴 시 메모리를 자동으로 통합합니다.
🔒
Multi-Layer DRM 3층 방어로 모델 디스틸레이션을 방지합니다. Zig 레벨 증명, 가짜 도구 주입, 서버측 추론 요약으로 구성됩니다.
