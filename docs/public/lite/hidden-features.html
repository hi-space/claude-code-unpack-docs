Feature Flags (기능 플래그)

44개의 피처 플래그는 컴파일 타임(공개 빌드에서 제거)과 런타임(GrowthBook 원격 제어)으로 분류됩니다.

피처 플래그의 의미

44개 플래그는 Claude Code의 진화 로드맵을 명시:
현재 상태: 반응형 단일 에이전트 코딩 어시스턴트
미래 상태: 자율적, 멀티모달(음성/텍스트), 멀티에이전트 개발 플랫폼
배포 전략: 컴파일 타임 플래그 12개는 공개 빌드에서 완전 제거 (dead code elimination). 런타임 플래그 15+개는 GrowthBook으로 Anthropic이 원격 제어 (재배포 불필요).

주의: KAIROS, VOICE_MODE, BUDDY, ULTRAPLAN은 디렉토리가 존재하지만 소스에서 inline feature() 호출로 게이트되지 않습니다. 대신 모듈 진입점(예: isVoiceGrowthBookEnabled())에서 다양한 게이팅 메커니즘을 사용합니다.

★ 모델 코드명

Model Codenames (모델 코드명)

유출된 소스코드에서 발견된 Claude 모델의 내부 코드네임. 공식적으로 문서화되지 않았지만 소스에 여러 곳에서 등장합니다.

코드네임 타입 상세
Capybara 프로젝트 코드네임 Sonnet v8 (1M 컨텍스트), "fast mode" 변형, 거짓 주장률 29-30%
Fennec 프로젝트 코드네임 Opus 4.6 전신, 마이그레이션 경로가 소스코드에 존재
Numbat 프로젝트 코드네임 차기 미출시 모델, 출시 일정이 소스코드에 임베딩됨
Tengu 피처 플래그 프리픽스 Claude Code 프로젝트 자체의 내부 이름, tengu_* 플래그와 텔레메트리에 사용 (모델 식별자 아님)

Capybara (Sonnet v8)

1. 1M 컨텍스트 윈도우

Capybara는 Sonnet v8의 내부 코드네임, 1M 토큰 컨텍스트 윈도우.
역할: Claude Code의 "fast mode" 구동 모델로 소스에서 발견.
트레이드오프: 거짓 주장률 29-30% (v4의 16.7% → 회귀). 속도 최적화로 인한 정확도 손실 시사.
용도: 반응성이 중요한 상황에서 속도 우선 선택 옵션.

Fennec (Pre-Opus 4.6)

2. 모델 마이그레이션 경로

Fennec은 Opus 4.6의 직계 전신 (공식 미공개).
발견 증거: 소스코드에 Fennec → Opus 마이그레이션 경로 존재 (resolveModelId 함수의 model mapping).
의미: Anthropic의 체계적 모델 업그레이드 프로세스 증명. 버전 전환 시 기존 설정/행동 호환성 유지 메커니즘.

Numbat (미출시 모델)

3. 공개적으로 발표되지 않은 차세대 모델

Numbat은 미발표 차세대 모델의 내부 코드네임.
발견:

  • 출시 일정이 소스코드에 embedded (feature flag gates 존재)
  • model_routing_logic에서 Numbat 조건부 분기
  • 다른 system prompt instructions (능력/제약 프로파일)
  • resolveModelId에서 "numbat" → 예약된 API ID 매핑
의미: Numbat은 이미 내부 개발 중이며 출시 준비 단계.

Tengu (프로젝트 코드네임 및 피처 플래그 프리픽스)

4. Claude Code 자체의 코드네임

Tengu는 모델이 아닌 Claude Code 프로젝트 자체의 내부 코드네임이자 피처 플래그 프리픽스. tengu_ 접두사가 모든 시스템에 일관적 네임스페이스 제공:

  • GrowthBook feature flags (tengu_anti_distill_fake_tool_injection 등)
  • OpenTelemetry 텔레메트리 이벤트
  • 환경 변수 (TENGU_DEBUG=1)
  • 설정 키 (tengu_log_tool_metrics)

효과: Claude Code 특정 기능을 다른 Anthropic 제품과 구분. Tengu는 모델 식별자가 아닌 기능 플래그 네임스페이스입니다.

Tengu 사용처

사용처 예시
피처 플래그 tengu_anti_distill_fake_tool_injection
텔레메트리 이벤트 tengu_ 접두사 분석 이벤트
GrowthBook 플래그 tengu_attribution_header

⚑ 미출시 기능

Unreleased Features (미출시 기능)

주요 컴파일 타임 플래그 뒤에 숨겨진 완전히 구현된 시스템들입니다. 각 기능은 별도 구조와 아키텍처를 가지고 있습니다.

1. KAIROS — 자율 데몬 모드

Compile-time Gated

가장 중요한 숨겨진 기능입니다. 완전히 구현되었지만 컴파일 타임 플래그로 비활성화되었습니다. <tick> 프롬프트 시스템으로 주기적으로 작업 상태를 평가하고, autoDream으로 유휴 시 메모리를 자동 통합합니다. KAIROS는 Claude Code가 완전히 자율적인 개발 데몬이 될 수 있음을 보여줍니다.

2. VOICE_MODE — Push-to-Talk 음성 인터페이스

Compile-time Gated

src/voice/ 디렉토리에 완전히 구현된 오디오 파이프라인:

  • 키보드 단축키로 마이크 활성화 (Alt+Space 또는 Cmd+Space)
  • voiceStreamSTT.ts를 통한 스트리밍 오디오 처리
  • Text-to-speech synthesis로 응답 전달
  • 웨이크 워드 및 명령 구문 인식

3. ULTRAPLAN — 원격 계획 세션 (최대 30분)

Compile-time Gated

현재 Plan Mode를 복잡한 아키텍처 결정을 위해 설계된 원격, 장기 실행 변형으로 확장합니다:

  • 로컬 vs 원격: UltraPlan은 Anthropic 인프라에서 원격 실행
  • 지속 시간: 최대 30분 (로컬은 2-5분)
  • 실행 방식: 여러 반복 호출로 문제를 반복적으로 분석
  • 용도: 마이크로서비스 아키텍처, 프레임워크 마이그레이션, 보안 강화

4. BUDDY — 터미널 펫 시스템

Compile-time Gated

가장 예상치 못한 발견입니다. Claude Code 옆에 나타나는 18종의 가상 반려동물:

  • 희귀도 시스템: Common (50%) → Uncommon (30%) → Rare (15%) → Legendary (5%)
  • 레벨 시스템: 완료된 작업을 통해 XP 획득
  • 업적 시스템: 마일스톤에 대한 특수 화장품 해금
  • 계절 이벤트: 제한된 시간 종 스폰 및 변형
  • 순수 장식: 핵심 기능에 영향 없음, 기능 플래그로 비활성화 가능

5. BRIDGE_MODE — Claude.ai 지속 연결

Compile-time Gated

Claude.ai에 대한 WebSocket 연결로 지속적인 백그라운드 세션과 원격 접근을 활성화합니다:

  • 지속적인 배경 세션: 웹 UI에서 시작하고 CLI/모바일에서 상태 확인
  • 원격 도구 실행: Claude.ai에서 파일 시스템 작업 또는 시스템 명령 실행
  • 크로스 플랫폼 연속성: 웹에서 시작 → CLI에서 계속 → 모바일에서 완료
  • 자동 재연결: Exponential backoff (1s → 30s max), 30초마다 heartbeat

6. WORKTREE_MODE — Git Worktree 격리

Compile-time Gated

Git worktree를 사용하여 안전한 기능 개발과 병렬 작업을 지원합니다:

  • 각 worktree는 .claude/worktrees/에 격리됨
  • 동시 코드 편집 시 충돌 없음
  • 독립적인 빌드, 테스트 실행 가능
  • tmux 세션 통합으로 병렬 작업 지원

7. PLAN_MODE — 읽기 전용 계획 도구

Compile-time Gated

계획 단계 중 우발적 실행을 방지하기 위한 읽기 전용 제약:

  • .claude/plans/ 디렉토리에 YAML frontmatter가 있는 Markdown 파일로 저장
  • 계획 중 Edit, Write, NotebookEdit, Bash 도구 차단
  • Read, Glob, Grep은 허용 (정보 수집)
  • 드래프트 → 검토 → 승인 → 실행 워크플로우

8. VIM_MODE — Vi/Vim 키바인딩

Compile-time Gated

Claude Code 터미널 입력에 포괄적인 Vi/Vim 키바인딩 추가:

  • 모달 편집: Normal (navigation) ↔ Insert ↔ Visual (selection)
  • 네비게이션: hjkl, w, b, ^, $, G, gg
  • 편집 작업: d, c, y, p, u, Ctrl-R
  • 커맨드 모드: :w, :q, :set, :%s///g
🔍
아키텍처 패턴 각 미출시 기능은 격리된 src/feature-name/ 디렉토리에 완전 구현되며, compile-time flag로 빌드 시 포함/제외.
배포 전략: 컴파일 타임 플래그 (KAIROS=true/false로 conditional compilation) + GrowthBook 런타임 제어 (이미 빌드된 기능의 on/off). 이 이중 계층으로 새 버전 배포 없이도 기능 제어 가능. 각 기능은 독립적 마이그레이션 경로, 롤백 가능.

추가 기능들

OpenTelemetry 통합

Datadog 또는 자체 호스팅 OTEL collector로 내보내지는 포괄적인 observability. PII-free diagnostic logging으로 프라이버시를 보호하면서 완전한 성능 추적을 제공합니다.

Desktop / Mobile / IDE 통합

Claude Code는 터미널을 넘어 확장됩니다:

  • Desktop: 윈도우 관리, 클립보드 통합, 전역 hotkeys
  • Mobile: iOS/Android companion app, 파일 공유, 음성 입력
  • Browser: Chrome Extension으로 GitHub PR inline review
  • IDE: VS Code 확장, JetBrains 플러그인
  • Apps: Slack bot, GitHub app

GrowthBook 원격 구성

단순 기능 플래그를 넘어 전체 원격 구성 및 긴급 제어를 제공합니다. Anthropic은 staged rollouts로 기능을 점진적 배포하고, 긴급 상황에 모든 installations에서 즉시 기능을 비활성화할 수 있습니다.

⚙ 설정 시스템

Configuration System (설정 시스템)

다층 설정 구조와 GrowthBook 원격 피처 플래그를 통한 동적 제어 시스템

1. GrowthBook Feature Flags

GrowthBook 클라이언트가 런타임에 기능 플래그를 평가합니다. tengu_ 접두사가 붙은 모든 플래그는 원격 구성에 대해 평가되며, Anthropic의 GrowthBook 대시보드의 변경 사항이 새 버전을 푸시하지 않고도 모든 Claude Code 설치 전체에 적용됩니다.

2. Configuration Layers

구성은 우선순위가 있는 여러 레이어를 통해 흐릅니다:

  • 1) CLI 플래그 (최우선)
  • 2) 환경 변수 (CLAUDE_CODE_*)
  • 3) 프로젝트 설정 (.claude/settings.json)
  • 4) 사용자 설정 (~/.claude/settings.json)
  • 5) GrowthBook 원격 플래그
  • 6) 컴파일된 기본값 (최하위)

3. Gradual Module Loading

Claude Code는 첫 응답 시간(TTFR)을 최소화하기 위해 빠른 경로 최적화를 사용합니다:

  • --version 및 --help: 0 모듈 로딩, 즉시 응답
  • 전체 트리 지연 로딩: REPL 모드와 대화형 세션에만 필요
이 최적화는 콜드 스타트에서 인지된 성능을 극적으로 향상시킵니다.
