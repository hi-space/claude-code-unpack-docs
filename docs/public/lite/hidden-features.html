44개 플래그는 Claude Code의 진화 로드맵을 명시합니다. 컴파일 타임 플래그 12개는 빌드할 때 포함/제외되며, 런타임 플래그 15+개는 GrowthBook으로 Anthropic이 재배포 없이 원격 제어합니다.

피처 플래그의 의미

44개 플래그는 Claude Code의 진화 로드맵을 명시합니다. 현재 상태: 반응형 단일 에이전트 코딩 어시스턴트, 미래 상태: 자율적, 멀티모달(음성/텍스트), 멀티에이전트 개발 플랫폼. 컴파일 타임 플래그 12개는 공개 빌드에서 dead code elimination으로 완전 제거되고, 런타임 플래그 15+개는 GrowthBook으로 원격 제어되어 재배포 불필요합니다.

★ 모델 코드명

Model Codenames (모델 코드명)

Claude 모델과 Claude Code 프로젝트의 내부 코드네임입니다. 소스 코드에 임베딩된 이들 이름은 Anthropic의 개발 로드맵과 모델 진화 과정을 드러냅니다.

코드네임 타입 상세
Capybara 프로젝트 코드네임 Sonnet v8 (1M 컨텍스트), "fast mode" 변형, 거짓 주장률 29-30%
Fennec 프로젝트 코드네임 Opus 4.6 전신, 마이그레이션 경로가 소스코드에 존재
Numbat 프로젝트 코드네임 차기 미출시 모델, 출시 일정이 소스코드에 임베딩됨
Tengu 피처 플래그 프리픽스 Claude Code 프로젝트 자체의 내부 이름, tengu_* 플래그와 텔레메트리에 사용 (모델 식별자 아님)

Capybara (Sonnet v8)

1. Fast Mode 변형 모델

Capybara는 Sonnet v8의 내부 코드네임으로 Claude Code의 "fast mode" 구동 모델입니다. 1M 토큰 컨텍스트 윈도우를 제공합니다. 속도 최적화의 트레이드오프로 거짓 주장률이 29-30% (Opus v4의 16.7% 대비) 상승했습니다. 반응성이 중요한 상황에서 정확도보다 속도를 우선하는 선택지입니다.

Fennec (Pre-Opus 4.6)

2. 모델 마이그레이션 경로

Fennec은 Opus 4.6의 직계 전신으로 공식 미공개 모델입니다. resolveModelId 함수에 Fennec → Opus 마이그레이션 경로가 존재하여 Anthropic의 체계적 모델 업그레이드 프로세스를 증명합니다. 버전 전환 시 기존 설정과 행동 호환성을 유지하는 메커니즘이 내장되어 있습니다.

Numbat (미출시 모델)

3. 공개적으로 발표되지 않은 차세대 모델

Numbat은 미발표 차세대 모델의 내부 코드네임입니다. 출시 일정이 소스코드 feature flag gates에 임베딩되어 있으며, model_routing_logic에서 조건부 분기가 존재합니다. 다른 system prompt instructions (능력/제약 프로파일)이 정의되어 있고, resolveModelId에서 "numbat" → 예약된 API ID 매핑이 존재합니다. Numbat은 이미 내부 개발 중이며 출시 준비 단계입니다.

Tengu (프로젝트 코드네임)

4. Claude Code 자체의 코드네임

Tengu는 모델이 아닌 Claude Code 프로젝트 자체의 내부 코드네임이자 피처 플래그 프리픽스입니다. tengu_ 접두사가 모든 시스템에 일관적 네임스페이스를 제공합니다: GrowthBook feature flags (tengu_anti_distill_fake_tool_injection 등), OpenTelemetry 텔레메트리 이벤트, 환경 변수 (TENGU_DEBUG=1), 설정 키 (tengu_log_tool_metrics). Tengu는 모델 식별자가 아닌 기능 플래그 네임스페이스로서 Claude Code를 다른 Anthropic 제품과 구분합니다.

⚑ 미출시 기능

Unreleased Features (미출시 기능)

완전히 구현되었지만 컴파일 타임 플래그로 비활성화된 기능들입니다. 각 기능은 별도 디렉토리에 구현되었으며, 재배포 없이 원격 플래그로 활성화될 준비가 되어 있습니다.

1. KAIROS — 자율 데몬 모드

Compile-time Gated

KAIROS는 사람의 요청 없이도 코딩 작업을 자동으로 수행하는 백그라운드 데몬입니다. <tick> 프롬프트 시스템으로 주기적으로 작업 상태를 평가하고, autoDream으로 유휴 시 메모리를 자동 통합합니다. 예약된 작업 실행, 자동 리베이스, PR 모니터링을 지원합니다. Claude Code가 완전히 자율적인 개발 데몬이 될 수 있음을 보여주는 가장 중요한 숨겨진 기능입니다.

2. VOICE_MODE — Push-to-Talk 음성 인터페이스

Compile-time Gated

src/voice/ 디렉토리에 완전히 구현된 오디오 파이프라인입니다. 키보드 단축키(Alt+Space 또는 Cmd+Space)로 마이크를 활성화하고, voiceStreamSTT.ts를 통한 스트리밍 오디오 처리, Text-to-speech synthesis로 응답 전달, 웨이크 워드 및 명령 구문 인식을 모두 지원합니다.

3. ULTRAPLAN — 30분 원격 계획 세션

Compile-time Gated

현재 Plan Mode를 복잡한 아키텍처 결정을 위해 설계된 원격, 장기 실행 변형으로 확장합니다. Anthropic 인프라에서 원격 실행되며 최대 30분 지속(로컬은 2-5분), 여러 반복 호출로 문제를 반복적으로 분석합니다. 마이크로서비스 아키텍처, 프레임워크 마이그레이션, 보안 강화 같은 복잡한 시나리오에 사용됩니다.

4. BUDDY — 18종 터미널 펫

Compile-time Gated

가장 예상치 못한 발견입니다. Claude Code 옆에 나타나는 18종의 가상 반려동물이 구현되어 있습니다. 희귀도 시스템(Common 50% → Uncommon 30% → Rare 15% → Legendary 5%), 레벨 시스템(완료된 작업을 통해 XP 획득)이 완전히 구현되었습니다. 순수 장식 기능이며 비활성화할 수 있습니다.

5. BRIDGE_MODE — Claude.ai WebSocket 연결

Compile-time Gated

Claude.ai와 WebSocket으로 양방향 연결하여 웹과 CLI 사이를 매끄럽게 넘나들 수 있습니다. 배경에서 지속적으로 실행되며 연결 끊김 시 자동으로 재연결됩니다. 두 인터페이스 간 작업 상태와 컨텍스트를 동기화합니다.

6. WORKTREE_MODE — Git Worktree 격리

Compile-time Gated

Git worktree를 통해 여러 작업을 동시에 진행할 수 있습니다. 각 작업은 격리된 워크트리에서 실행되어 브랜치 충돌 없이 병렬로 처리 가능합니다. 로컬 상태 관리와 복수 작업 추적 기능이 구현되어 있습니다.

7. PLAN_MODE — 읽기 전용 계획 도구

Compile-time Gated

계획 단계에서 모든 실행 도구를 차단하고 읽기 전용 동작만 허용합니다. 사용자가 계획을 검토한 후 명시적 승인을 거쳐야만 실행 도구에 접근할 수 있습니다. 무분별한 자동 실행을 방지하는 안전장치입니다.

8. VIM_MODE — Vi/Vim 키바인딩

Compile-time Gated

Claude Code의 CLI 인터페이스에서 Vi/Vim 키바인딩을 완전히 지원합니다. 노멀 모드, 인서트 모드, 커맨드 모드, 비주얼 모드 같은 Vim의 모든 주요 모드와 j/k/h/l 네비게이션, w/b/e 단어 이동, / 검색, :w/:q 커맨드를 구현했습니다.

🔍
아키텍처 패턴 각 기능은 별도 디렉토리에 구현되고, 빌드 시 포함하거나 제외합니다. 플래그와 원격 제어로 새 버전 배포 없이도 기능을 켜거나 끌 수 있습니다.

추가 기능들

OpenTelemetry 통합

성능을 모니터링하고 진단 정보를 수집합니다. tengu_otel_sampling_rate 플래그로 샘플링 비율을 동적으로 조정할 수 있으며, 성능 저하 없이 상세한 텔레메트리 데이터를 수집합니다. 개인정보는 철저히 보호됩니다.

Desktop / Mobile / IDE 통합

DESKTOP_MOBILE_IDE_INTEGRATION 플래그로 데스크톱, 모바일, IDE, 브라우저 등 다양한 플랫폼에서 Claude Code를 사용할 수 있습니다. tengu_ide_integration_level로 CLI 전용 또는 IDE 포함 모드를 런타임에 선택합니다. 크로스 플랫폼 작업 동기화 인프라가 구현되어 있습니다.

GrowthBook 원격 구성

15+ 런타임 플래그를 GrowthBook으로 Anthropic이 원격 제어하여 새 버전 배포 없이 기능을 켜거나 끕니다. 단계적 롤아웃(tengu_bridge_mode_rollout), 긴급 비활성화(tengu_distillation_protection off), A/B 테스트(tengu_classifier_sensitivity 0.5-1.0 범위) 등을 지원합니다.

⚙ 설정 시스템

Configuration System (설정 시스템)

CLI 플래그, 환경 변수, 프로젝트/사용자 설정, 원격 플래그, 기본값의 계층적 구조로 기능을 제어합니다. 빌드 타임 컴파일과 런타임 제어를 모두 지원합니다.

1. GrowthBook Feature Flags

실행 중에 GrowthBook에서 기능 플래그를 동적으로 확인합니다. 새 버전을 배포하지 않고도 모든 설치에 즉시 변경사항을 적용하며, 지역별/세그먼트별 차등 적용도 가능합니다. 롤백은 단 한 번의 서버 설정 변경으로 즉시 반영됩니다.

2. Configuration Layers

설정은 다층 우선순위를 따릅니다: CLI 플래그 (최고) → 환경 변수 → 프로젝트 설정 → 사용자 설정 → 원격 플래그 → 기본값 (최저). 각 계층은 상위 계층이 설정하지 않은 항목만 적용되어 유연한 오버라이드를 지원합니다.

3. Gradual Module Loading

빠른 응답을 위해 필수 모듈만 먼저 로드하고, 사용 시점에 나머지를 로드합니다(lazy loading). claude-code version 같은 간단한 명령은 100ms 이내에 응답하고, 복잡한 기능은 필요시에만 로드되어 메모리 효율성과 반응성을 동시에 확보합니다.
