CLI / UI Architecture

React 기반 터미널 렌더러로 화면을 실시간 갱신하고, 병렬 사전 로딩으로 초기 준비를 가속화하며, 3가지 최적화된 명령 유형으로 모든 작업을 신속하게 처리합니다.

⚡ Features

Core CLI / UI Features

Claude Code는 커스텀 React 렌더러로 터미널 화면을 실시간 업데이트하고, 효율적인 로딩과 병렬 처리로 빠른 시작을 제공하며, 최적화된 명령 분류 시스템으로 모든 작업을 신속하게 처리합니다.

1. Custom Fiber Reconciler

약 1,722줄의 커스텀 렌더링 엔진이 JSX 컴포넌트를 터미널용 형식으로 변환합니다. 렌더링 파이프라인: React 컴포넌트 트리 분석 → 상태 변경 감지 → 가상 중간 표현으로 변환 → 터미널 제어 명령 생성 → 화면에 출력. React의 useState, useEffect 등 익숙한 패턴을 터미널 환경에 적응시켰으며, 동기 커밋 기법으로 내부 상태와 화면 표시 간의 불일치를 방지합니다.

2. Progressive Module Loading

CLI 진입점에서 --version이나 --help, --list, --status 같은 단순 정보 요청을 감지하면 모든 모듈 로딩 단계를 완전히 건너뛰고 즉시 응답합니다(0ms 추가 대기). 반대로 대화형 REPL 세션이나 복잡한 대화 기능이 필요한 경우만 전체 시스템을 지연 로드하기 때문에, 빈번한 정보 조회나 버전 확인 작업에서 불필요한 모듈 초기화 오버헤드를 완전히 제거할 수 있습니다. 이 전략은 스크립트 환경에서 반복되는 호출이나 CI/CD 파이프라인에서 특히 중요하여, 전체 워크플로우 실행 시간을 획기적으로 단축하고 개발자 생산성을 크게 향상시킵니다.

3. Parallel Prefetching

REPL 시작 시 애플리케이션 설정 파일 로드(약 200ms), 암호화된 자격증명 저장소 조회(약 500ms), MCP 서버 초기화(약 300ms) 같은 3가지 독립적 작업을 Promise.all을 사용한 비동기 병렬 처리로 동시 실행합니다. 순차 처리 방식으로는 약 1000ms(1초)가 필요하지만 병렬 처리로는 약 500ms로 단축되어, 사용자가 체감할 수 있는 2배의 속도 개선을 제공합니다. 특히 시작 프로파일러가 각 단계의 소요 시간을 자동으로 측정하여 성능 병목을 식별하고, 지속적인 최적화와 성능 모니터링을 가능하게 합니다.

4. Raw Mode Synchronicity

Ctrl+C, Escape, 또는 마우스 클릭 같은 특수 키 이벤트는 React의 렌더링 후 동기 커밋 단계에서 처리되어 터미널 raw mode 상태 변경이 React 컴포넌트 업데이트와 정확히 동기화됩니다. 일반적인 비동기 처리 대신 동기 처리를 사용함으로써 터미널 제어 상태와 React 내부 상태 사이의 불일치로 인한 예측 불가능한 동작이나 타이밍 이슈를 철저히 방지합니다. 이는 시그널 인터럽트 처리에서 특히 중요하여, 신호 처리의 정확성과 응답성을 동시에 보장하고 터미널 세션의 안정성을 크게 향상시킵니다.

5. 3 Command Types

모든 명령어는 처리 방식에 따라 세 가지로 분류 및 최적화됩니다:
- prompt: 사용자의 자연어 입력을 언어 모델 API에 전송하여 깊이 있는 분석과 생성 결과를 제공합니다.
- local: /help, /clear, /list 같은 슬래시 명령을 즉시 실행하여 0ms 지연으로 빠른 응답을 보장합니다.
- local-jsx: /config 같은 설정 대화를 터미널에서 React 컴포넌트로 렌더링하여 대화형 UI를 제공합니다.
각 작업의 특성에 맞는 실행 경로를 선택하여, 깊이 있는 분석과 빠른 응답을 동시에 지원합니다.

💻 Bootstrap

Bootstrap & Rendering Pipeline

CLI 진입점부터 터미널 출력까지 각 단계를 최적화하여, 간단한 명령은 즉시 처리하고 복잡한 작업은 병렬 로딩으로 가속화하며, React 컴포넌트를 터미널 화면으로 변환하는 효율적인 렌더링 파이프라인입니다.

Fast Path 분기

CLI 진입점에서 첫 번째 작업은 들어온 명령의 타입을 빠르게 분석하는 것입니다. --version이나 --help, --list 같은 단순 정보 요청을 감지하면 모듈 로딩 단계를 완전히 건너뛰고 즉시 결과를 출력합니다(0ms 추가 대기). 반대로 REPL 모드나 대화형 세션이 필요한 경우는 필요한 전체 시스템을 준비합니다. 이 전략으로 빈번한 버전 확인이나 정보 조회 작업에서 불필요한 초기화 오버헤드를 제거하여, 사용자 경험을 크게 개선하고 스크립트 자동화 시나리오에서 전체 실행 시간을 단축할 수 있습니다.

병렬 사전 로딩

REPL 세션이 시작될 때, 시스템은 여러 초기화 작업을 순차가 아닌 동시에 시작합니다:
- 애플리케이션 설정 파일 로드 (약 200ms)
- 암호화된 자격증명 저장소 네트워크 조회 (약 500ms)
- MCP 서버 초기화 및 핸드셰이크 (약 300ms)
이 세 작업이 Promise.all을 통해 병렬 실행됩니다. 만약 순차적으로 처리했다면 약 1000ms(1초)가 소요되지만, 병렬 처리로는 가장 느린 작업 기준 약 500ms로 단축되어 전체 대기 시간을 2배 감소시킵니다. 내장된 성능 프로파일러가 각 작업의 정확한 소요 시간을 자동으로 측정하고 로깅하여, 성능 병목을 식별하고 지속적인 최적화와 성능 모니터링을 가능하게 합니다.

Ink Reconciler 파이프라인

약 1,722줄의 커스텀 렌더링 엔진이 다음 5단계로 작동합니다:
(1) React 컴포넌트 트리 분석
(2) 상태 변경 감지
(3) 가상 중간 표현으로 변환
(4) 터미널 제어 명령으로 컴파일
(5) 최적화된 형식으로 화면 출력
덕분에 개발자는 익숙한 React 패턴(useState, useEffect)을 그대로 사용하면서 터미널에 최적화된 인터페이스를 구축할 수 있습니다.

🟢 State

useInput 키 캡처

React 훅으로 제공되는 useInput을 통해 각 컴포넌트는 독립적으로 키보드 입력 이벤트를 구독하고 처리할 수 있습니다. 사용자가 키를 누르면 입력 값은 먼저 해당 컴포넌트의 로컬 상태에 반영되고, 필요에 따라 앱 전역 상태 저장소(Redux, Zustand 등)로 전파됩니다. 이 단방향 데이터 흐름 아키텍처로 모든 UI 요소가 항상 일관된 상태를 바탕으로 렌더링되므로, 여러 입력 필드가 있는 복잡한 터미널 인터페이스에서도 상태 일관성을 완벽하게 보장하고 예측 불가능한 UI 동작을 방지합니다.

useLayoutEffect 동기 커밋

Ctrl+C, Escape, 또는 특수 키 조합 같은 중요한 인터럽트 이벤트는 React의 렌더링 후 DOM 업데이트 직전 단계에서 실행되는 동기 훅을 통해 처리됩니다. 일반적인 비동기 처리를 사용하지 않기 때문에 마이크로태스크 큐에 의한 지연이 완전히 제거되어, 터미널 제어 상태(raw mode 활성/비활성화 등) 변경이 React 컴포넌트 상태 업데이트와 정확히 동기화됩니다. 이를 통해 터미널 제어 상태와 React 컴포넌트 상태 사이의 불일치로 인한 예측 불가능한 동작이나 타이밍 이슈를 철저히 방지하여, 신호 처리의 정확성과 사용자 응답성을 동시에 완벽하게 보장합니다.

Ink DOM 차이 비교

React의 강력한 가상 DOM 재조정 알고리즘을 터미널 렌더링에 맞게 적용하여, Ink DOM이라 불리는 터미널용 가상 표현이 각 렌더링 사이클마다 생성됩니다. 시스템은 이전 렌더링 결과(이전 Ink DOM)와 현재 렌더링 결과(현재 Ink DOM)를 세밀하게 비교하고, 실제로 변경된 부분만 계산하여 차이점 목록을 만듭니다. 전체 터미널 화면을 매번 다시 그리는 비효율적인 방식 대신 변경사항만 터미널에 출력하므로, 불필요한 제어 명령 전송을 제거하고 화면 깜빡임과 깜박거림을 최소화하며, 네트워크 대역폭 사용량과 렌더링 성능을 획기적으로 개선합니다.
