⚡ Features

Core CLI / UI Features

Claude Code의 터미널 렌더링 및 명령 시스템의 5가지 핵심 특성

1. Custom Fiber Reconciler

커스텀 React Fiber 재조정자(약 1,722줄 Ink 엔진)는 HTML DOM 대신 ANSI escape code를 생성합니다. 아키텍처는 다음과 같습니다: JSX → Reconciler → ANSI Escape Sequences → Terminal Output. useState/useEffect를 터미널 렌더링에 맞게 적응시키고, useLayoutEffect (동기 커밋 페이즈)로 터미널/React 상태 불일치를 방지합니다.

2. Progressive Module Loading

CLI 진입점 검사: --version, --help → 즉시 출력, 모듈 로딩 0ms. REPL/대화형 세션만 전체 모듈 트리 lazy loading. 효과: 첫 응답 시간(TTFR)을 밀리초 단위로 최적화합니다. 버전 확인 같은 빠른 작업은 콜드 스타트 페널티 없습니다.

3. Parallel Prefetching

REPL 시작 시 3가지 독립적 작업을 Promise.all로 병렬화합니다:

  • MDM 구성 로드 (~200ms)
  • Keychain 자격증명 조회 (~500ms)
  • MCP 서버 초기화 (~300ms)
순차 처리 ~1000ms → 병렬 처리 ~500ms (3배 개선). 시작 프로파일러가 각 단계를 측정하여 병목을 식별합니다.

4. Raw Mode Synchronicity

터미널의 raw mode 상태 변경은 useLayoutEffect를 통해 React의 commit phase에서 동기적 실행됩니다. useEffect와 달리 터미널 상태와 React 렌더링 상태의 불일치를 방지합니다. Ctrl+C 시그널 처리에서 특히 중요합니다.

5. 3 Command Types

각 명령 유형별 최적화된 실행 경로:

  • prompt — 모델 API 호출 (대부분 사용자 입력)
  • local — JavaScript 함수 직실행 (/clear, /help 등, 0 latency)
  • local-jsx — React 컴포넌트 로컬 렌더링 (설정 대화 등)
이 분류로 유저 프롬프트만 API 호출하고, 슬래시 명령/UI는 로컬 처리로 즉각 응답성을 제공합니다.

Fast Path 분기

CLI 진입점에서 --version, --help 같은 단순 명령은 모듈 로딩 없이 즉시 출력합니다. REPL/대화형 세션만 전체 모듈 트리를 lazy loading하므로, 빈번한 버전 확인이나 도움말 호출에서 콜드 스타트 페널티가 발생하지 않습니다.

병렬 사전 로딩

REPL 시작 시 MDM 구성 로드(~200ms), Keychain 자격증명 조회(~500ms), MCP 서버 초기화(~300ms)를 Promise.all로 병렬 실행합니다. 순차 처리 시 ~1000ms가 소요되는 작업을 ~500ms로 단축하며, 시작 프로파일러가 각 단계의 소요 시간을 측정하여 병목 지점을 식별합니다.

Ink Reconciler 파이프라인

커스텀 React Fiber 재조정자(약 1,722줄)가 JSX 컴포넌트를 HTML DOM 대신 ANSI escape code로 변환합니다. React 컴포넌트 트리 → Ink 재조정자 → 가상 Ink DOM → ANSI 코드 생성 → 터미널 출력 순서로 처리되며, useState/useEffect 같은 React 훅을 터미널 렌더링에 맞게 적응시킵니다.

useInput 키 캡처

키보드 입력은 useInput 훅을 통해 React 상태로 변환됩니다. 각 컴포넌트가 독립적으로 키 이벤트를 구독할 수 있으며, 입력 값은 컴포넌트 로컬 상태를 거쳐 앱 전역 스토어로 전파됩니다. 이 단방향 데이터 흐름이 복잡한 터미널 UI에서도 상태 일관성을 보장합니다.

useLayoutEffect 동기 커밋

Ctrl+C, Escape 같은 특수 키 이벤트는 useLayoutEffect를 통해 React commit phase에서 동기적으로 처리됩니다. 비동기 useEffect와 달리 터미널의 raw mode 상태 변경이 렌더링과 정확히 동기화되어, 터미널 상태와 React 상태 사이의 불일치를 방지합니다. 시그널 처리의 정확성에 핵심적인 설계입니다.

Ink DOM 차이 비교

React의 가상 DOM 재조정과 유사하게, Ink DOM은 이전 렌더링 결과와 현재 결과의 차이만 계산하여 변경된 ANSI 시퀀스만 터미널에 출력합니다. 전체 화면을 다시 그리는 대신 변경분만 패치하므로 터미널 깜빡임을 최소화하고 렌더링 성능을 최적화합니다.
