лҸ„кө¬ мӢңмҠӨн…ң к°ңмҡ”

Claude Codeмқҳ лҸ„кө¬ мӢңмҠӨн…ңмқҖ 23+ л№ҢнҠёмқё лҸ„кө¬мҷҖ MCP нҶөн•©мқ„ нҶөн•ҙ нҢҢмқј м ‘к·ј, мҪ”л“ң мӢӨн–ү, мӣ№ кІҖмғү л“ұ лӢӨм–‘н•ң кё°лҠҘмқ„ м ңкіөн•©лӢҲлӢӨ.

н•өмӢ¬ мқёмӮ¬мқҙнҠё

1. Fail-Closed Defaults

лӘЁл“  мғҲлЎңмҡҙ лҸ„кө¬лҠ” кё°ліём ҒмңјлЎң кұ°л¶Җ/мҲңм°Ё лӘЁл“ңлЎң мғқм„ұлҗ©лӢҲлӢӨ. к°ңл°ңмһҗк°Җ лӘ…мӢңм ҒмңјлЎң к¶Ңн•ңкіј лҸҷмӢңм„ұмқ„ нҷңм„ұнҷ”н•ҙм•ј н•ҳлҜҖлЎң, м„Өм •мқ„ л№јлЁ№м–ҙлҸ„ мһҗлҸҷмңјлЎң к°ҖмһҘ м•Ҳм „н•ң мғҒнғңк°Җ лҗ©лӢҲлӢӨ. мқҙ 'fail-closed' мӣҗм№ҷмқҙ ліҙм•Ҳ мӮ¬кі лҘј мӣҗмІң м°ЁлӢЁн•ҳлҠ” н•өмӢ¬ м„Өкі„ мІ н•ҷмһ…лӢҲлӢӨ.

2. 3-Layer Permission Check

лҸ„кө¬ мӢӨн–ү м „ 3лӢЁкі„ к¶Ңн•ң кІҖмҰқмқҙ мҲҳн–үлҗ©лӢҲлӢӨ: мһ…л Ҙ мҠӨнӮӨл§Ҳ кІҖмҰқ, лҸ„кө¬лі„ м»ӨмҠӨн…Җ к¶Ңн•ң нҷ•мқё, м „м—ӯ к¶Ңн•ң м •мұ… м Ғмҡ©. к°Ғ л Ҳмқҙм–ҙк°Җ лҸ…лҰҪм ҒмңјлЎң кұ°л¶Җн•  мҲҳ мһҲм–ҙ мӢ¬мёө л°©м–ҙ(defense-in-depth)лҘј кө¬нҳ„н•ҳкі , н•ң лӢЁкі„мқҳ м„Өм • мӢӨмҲҳк°Җ м „мІҙ мӢңмҠӨн…ңмқ„ мң„нҳ‘н•ҳм§Җ м•ҠлҸ„лЎқ н•©лӢҲлӢӨ.

3. Deferred Tool Loading

мһҗмЈј мӮ¬мҡ©н•ҳм§Җ м•ҠлҠ” лҸ„кө¬лҠ” мҙҲкё° н”„лЎ¬н”„нҠём—җ м „мІҙ мҠӨнӮӨл§ҲлҘј нҸ¬н•Ён•ҳм§Җ м•Ҡкі , мқҙлҰ„л§Ң л…ём¶ңн•©лӢҲлӢӨ. лӘЁлҚёмқҙ н•ҙлӢ№ лҸ„кө¬лҘј мӮ¬мҡ©н•ҳл Ө н•  л•Ң лҸҷм ҒмңјлЎң мҠӨнӮӨл§ҲлҘј лЎңл“ңн•ҳлҜҖлЎң, мҙҲкё° н”„лЎ¬н”„нҠё нҒ¬кё°лҘј м•Ҫ 3-5K нҶ нҒ° м Ҳк°җн•ҳм—¬ мқ‘лӢө мҶҚлҸ„мҷҖ л№„мҡ© нҡЁмңЁмқ„ к°ңм„ н•©лӢҲлӢӨ.

4. Result Persistence

лҸ„кө¬ мӢӨн–ү кІ°кіјк°Җ нҒ° кІҪмҡ° м „мІҙ кІ°кіјлҘј м»Ён…ҚмҠӨнҠём—җ л„Јм§Җ м•Ҡкі  л””мҠӨнҒ¬м—җ м ҖмһҘн•©лӢҲлӢӨ. лӘЁлҚём—җкІҢлҠ” мҡ”м•Ҫ(preview)л§Ң м „лӢ¬н•ҳм—¬ м»Ён…ҚмҠӨнҠё мңҲлҸ„мҡ°лҘј нҡЁмңЁм ҒмңјлЎң мӮ¬мҡ©н•©лӢҲлӢӨ. нҒ° нҢҢмқј мқҪкё°, л°©лҢҖн•ң кІҖмғү кІ°кіј, кёҙ м…ё м¶ңл Ҙ л“ұм—җм„ң нҠ№нһҲ мӨ‘мҡ”н•ң м „лһөмһ…лӢҲлӢӨ.

5. Prompt Cache Stability

лҸ„кө¬ лӘ©лЎқмқ„ APIм—җ м „лӢ¬н•  л•Ң н•ӯмғҒ м •л ¬лҗң мҲңм„ңлҘј мң м§Җн•©лӢҲлӢӨ. Anthropic APIмқҳ prompt cachingмқҖ лҸ„кө¬ лӘ©лЎқмқҳ мҲңм„ңк°Җ л°”лҖҢл©ҙ мәҗмӢң нӮӨк°Җ лӢ¬лқјм ё нһҲнҠёмңЁмқҙ л–Ём–ҙм§ҖлҜҖлЎң, лҸҷмқјн•ң лҸ„кө¬ м„ёнҠёлҠ” н•ӯмғҒ лҸҷмқјн•ң мәҗмӢң нӮӨлҘј мғқм„ұн•ҳлҸ„лЎқ ліҙмһҘн•©лӢҲлӢӨ. м „мІҙ лҸ„кө¬ мҠӨнӮӨл§ҲлҠ” м•Ҫ 14-17K нҶ нҒ°мқ„ м°Ём§Җн•©лӢҲлӢӨ.
