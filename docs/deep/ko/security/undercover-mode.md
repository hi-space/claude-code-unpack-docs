# 언더커버 모드

언더커버 모드는 외부 또는 공개 컨텍스트에서 동작할 때 Claude Code의 출력에서 모든 내부 Anthropic 메타데이터를 제거하는 보안 메커니즘이다.

## 목적

Claude Code가 공개 또는 오픈소스 레포지토리에서 동작할 때, 다음의 우발적 노출을 방지한다:

- 내부 모델 코드네임 (Capybara, Fennec, Numbat, Tengu)
- 미출시 모델 버전 번호
- 내부 레포지토리 이름
- 내부 툴링 및 인프라 상세
- Slack 채널 이름
- 내부 단축 링크

## 활성화

| 방법 | 상세 |
|------|------|
| 자동 | 외부 빌드에서 기본 활성 |
| 환경 변수 | `CLAUDE_CODE_UNDERCOVER=1`로 강제 활성화 |
| 코드 제거 | 외부 빌드에서 함수가 데드코드 제거되어 trivial return |

::: warning
외부 빌드에서는 언더커버 모드를 **비활성화할 방법이 없다**. 관련 코드 경로가 컴파일 타임에 제거되어, 항상 살균된 출력을 반환하는 no-op 함수로 축소된다.
:::

## 활성화 로직

언더커버 모드는 **안전 우선 기본값**을 따른다: 현재 레포지토리가 내부 레포지토리로 명시적으로 인식되지 않는 한 활성화된다.

- **강제 ON**: `CLAUDE_CODE_UNDERCOVER=1`을 설정하면 레포지토리 컨텍스트와 관계없이 언더커버 모드 활성화
- **자동 감지 (기본값)**: 시스템이 레포지토리의 원격 URL을 내부 허용 목록과 대조 확인. 원격이 허용 목록과 매치되지 않으면 언더커버 모드 자동 활성화
- **강제 OFF 불가**: 외부 빌드에서는 언더커버 모드를 비활성화할 방법이 없다. 코드 경로가 컴파일 타임에 제거됨

이 설계는 기본 상태가 항상 안전하도록 보장한다. 사용자가 설정을 잊어버렸더라도 언더커버 모드는 우발적 정보 누수로부터 보호한다.

### 내부 레포지토리 감지

시스템은 원격 URL을 기반으로 레포지토리를 3가지 카테고리로 분류한다:

| 분류 | 동작 |
|-----|------|
| **내부** | 언더커버 모드 OFF - 출력에 완전한 메타데이터 |
| **외부** | 언더커버 모드 ON - 메타데이터 제거 |
| **알 수 없음** (원격 없음) | 언더커버 모드 ON - 안전한 기본값 |

분류 결과는 세션 지속 시간 동안 캐시되어 반복 검사를 피한다.

## 제거되는 것

언더커버 모드가 활성화되면 시스템 프롬프트에 지시문이 주입된다:

> "You are operating UNDERCOVER in a PUBLIC/OPEN-SOURCE repository"

이는 모델이 다음을 언급하지 않도록 트리거한다:

| 카테고리 | 예시 |
|---------|------|
| 모델 코드네임 | Capybara, Fennec, Numbat, Tengu |
| 버전 번호 | 내부 모델 반복 번호 |
| 레포지토리 이름 | 내부 Anthropic 레포 참조 |
| 인프라 | 내부 툴링, 서비스, 엔드포인트 |
| 통신 | Slack 채널, 내부 링크 |

## 코드네임 마스킹

언더커버 모드가 활성화되면, 내부 모델 코드네임이 모든 보이는 출력에서 마스킹된다. 처음 3글자만 표시되고 나머지는 별표로 교체된다:

| 원본 | 마스킹됨 |
|-----|---------|
| `capybara-v2-fast` | `cap*****-v2-fast` |
| `fennec-preview` | `fen*****-preview` |

이 마스킹은 상태 표시, 로그 출력, 사용자에게 보이거나 터미널 출력에 캡처될 수 있는 모든 모델 식별 문자열에 적용된다.

## 커밋 기여도

외부 레포지토리에서 동작할 때, 언더커버 모드는 생성된 커밋 및 풀 요청에서 내부 기여도를 제거한다:

- **Co-authored-by 라인** 내부 식별자 참조 제거
- **커밋 메시지** 제출 전 내부 참조 확인
- 시스템이 커밋 및 PR 생성 프롬프트에 추가 안전 지시문을 주입

### 처음 공개 시 알림

언더커버 모드가 자동 감지로 활성화될 때 (환경 변수가 아님), 일회성 알림 대화가 사용자에게 동작을 설명한다. 이 알림은:

- 사용자당 한 번만 표시 (사용자 설정의 영구 플래그를 통해 추적)
- 환경 변수로 활성화가 강제될 때 억제 (사용자가 명시적으로 옵트인)
- 메타데이터가 제거되는 이유와 사용자 워크플로우에 어떤 의미인지 설명

## 설계 철학

언더커버 모드는 정보 누수에 대한 **다층 방어** 접근을 반영한다:

1. **빌드 타임 제거**: 외부 빌드에서 관련 코드 경로가 데드코드 제거되어 trivial return로 축소된다. 이는 런타임 토글이 아니다—코드가 바이너리에 존재하지 않는다.

2. **런타임 안전망**: 내부 빌드가 외부 컨텍스트에서 동작할 때, 런타임 검사가 추가 계층을 제공한다. 개발자가 실수로 공개 레포지토리에서 내부 빌드를 사용하더라도 언더커버 모드가 자동 활성화된다.

3. **프롬프트 레벨 강제**: 활성화되면 시스템 프롬프트에 내부 상세를 언급하지 않도록 지시하는 지시문이 포함된다. 이는 기술적 제어 위에 동작 계층을 만든다.

빌드 타임, 런타임, 프롬프트 레벨 제어의 조합은 정보 누수 보호에 단일 실패점이 없음을 의미한다.
